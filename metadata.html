Recording Metadata¶

Usage Example¶

Metadata is bundled with the data generated by a scan. It can be used by callbacks (e.g., to tell plots how to label their axes) or simply used to identify and analyze the data later.

In [1]: gs.RE.md['project'] = 'my xray project'

In [2]: gs.RE.md['sample'] = {'color': 'red', 'dimensions': [10, 20, 5]}

Or specified when the scan is run.

In [3]: ct(experimenter='Emily', mood='excited')
+------------+-------------------+
|   seq_num  |             time  |
+------------+-------------------+
|         1  |  15:42:35.171504  |
+------------+-------------------+
Out[3]: ['e814230d-d1c2-4145-9180-5fa5aee71800']

Note

Structured data, such as

{'color': 'red', 'dimensions': [10, 20, 5]}

is much better than a long string like

'red_10_20_5'

because it is searchable and self-describing. To encourage good practices, the RunEngine inists that ‘sample’ be a dictionary. Any other fields you invent can be anything you want.

Persistence Between Scans¶

The following fields are automatically reused between runs unless overridden.

  • sample
  • project
  • owner
  • group
  • beamline_id
  • config (which should rarely change; see below)
  • scan_id (which is automatically incremented)

Custom fields, like ‘experimenter’ and ‘mood’ in the example above, are not reused by default, as we can see below.

In [4]: ct()
+------------+-------------------+
|   seq_num  |             time  |
+------------+-------------------+
|         1  |  15:42:35.369579  |
+------------+-------------------+
Out[4]: ['26b6d104-b936-4d83-8fcf-51b5c71477e1']

In [5]: ct(sample={'color': 'blue', 'dimensions': [3, 1, 4]})
+------------+-------------------+
|   seq_num  |             time  |
+------------+-------------------+
|         1  |  15:42:35.550158  |
+------------+-------------------+
Out[5]: ['1acf3880-17ea-4ad3-95d2-6f9a15dd4ddd']

To add a custom field to the list of peristent fields, use RE.persistent_fields.append('experimenter'). Use RE.persistent_fields.remove('experimenter') to stop persisting it. Fields that are required by our Document specification—owner, group, beamline_id, and config—cannot be removed. (More on these below.)

To review the metadata before running ascan, check gs.RE.md, which behaves like a Python dictionary.

In [6]: gs.RE.md['sample']
Out[6]: {'color': 'blue', 'dimensions': [3, 1, 4]}

To start fresh:

In [7]: gs.RE.md.clear()

Required Fields¶

Some fields and required by our Document specification, and the RunEngine will raise a KeyError if they are not set. These fields are:

  • owner
  • group
  • beamline_id (e.g., ‘csx’)
  • config, a dictionary describing the hardware, calibration, dead pixels on detectors, etc.

standard_config.py fills some of these in automatically (e.g., ‘owner’ defaults to the username of the UNIX user currently logged in).
