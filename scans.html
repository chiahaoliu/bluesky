Object-Oriented Scan Interface¶

In the object-oriented interface, the scan object is not bundled with a copy of the RunEngine. That is, the instructions and the execution are treated separately.

Each of the scans in the Simple Scan Interface is backed by a Scan object. Once an instance of a Scan is created, it can be reused without respecifying its parameters.

Usage Example¶

Make an instance of a scan, passing in parameters.

In [1]: from bluesky.scans import AbsScan

In [2]: my_scan = AbsScan([det1, det2], motor, 1, 5, 10)

Running the scan is a separate step. The same scan can be run multiple times without respecifying its parameters.

RE(my_scan) RE(my_scan)

Any of the scan’ss parameters can be updated individually.

In [3]: my_scan.num = 4  # change number of data points from 10 to 4

In [4]: RE(my_scan)
Out[4]: ['da095905-1742-4021-a5f8-02d67e13c5c5']

In [5]: my_scan.detectors.append(det3)  # add another detector

In [6]: RE(my_scan)
Out[6]: ['8e3abb01-c3b5-4dad-a554-8ea6baa5fa44']

The set method is a convenient way to update multiple parameters at once.

In [7]: my_scan.set(start=20, stop=25)

Count¶

bluesky.scans.Count(detectors, num=1, delay=0)¶

Take one or more readings from the detectors. Do not move anything.

Parameters:
  • detectors (list) – list of ‘readable’ objects
  • num (integer, optional) – number of readings to take; default is 1
  • delay (float) – time delay between successive readings; default is 0

Examples

Count three detectors.

>>> c = Count([det1, det2, det3])
>>> RE(c)

Count them five times with a one-second delay between readings.

>>> c = Count([det1, det2, det3], 5, 1)
>>> RE(c)

Absolute Scans¶

bluesky.scans.AbsScan(*args, **kwargs)¶

Absolute scan over one variable in equally spaced steps

Parameters:
  • detectors (list) – list of ‘readable’ objects
  • motor (object) – any ‘setable’ object (motor, temp controller, etc.)
  • start (float) – starting position of motor
  • stop (float) – ending position of motor
  • num (int) – number of steps

Examples

Scan motor1 from 0 to 1 in ten steps.

>>> my_scan = AbsScan(motor1, [det1, det2], 0, 1, 10)
>>> RE(my_scan)
# Adjust a Parameter and run again.
>>> my_scan.num = 100
>>> RE(my_scan)

bluesky.scans.OuterProductAbsScan(detectors, *args)¶

Absolute scan over a mesh; each motor is on an independent trajectory

Parameters:
  • detectors (list) – list of ‘readable’ objects
  • start1, stop1, num1, motor2, start2, stop2, num2, snake2, ..., motorN, startN, stopN, numN, snakeN (motor1,) – motors can be any ‘setable’ object (motor, temp controller, etc.) Notice that the first motor is followed by start, stop, num. All other motors are followed by start, stop, num, snake where snake is a boolean indicating whether to following snake-like, winding trajectory or a simple left-to-right trajectory.

Adative Scans¶

bluesky.scans.AdaptiveAbsScan(*args, **kwargs)¶

Absolute scan over one variable with adaptively tuned step size

Parameters:
  • detectors (list) – list of ‘readable’ objects
  • target_field (string) – data field whose output is the focus of the adaptive tuning
  • motor (object) – any ‘setable’ object (motor, temp controller, etc.)
  • start (float) – starting position of motor
  • stop (float) – ending position of motor
  • min_step (float) – smallest step for fast-changing regions
  • max_step (float) – largest step for slow-chaning regions
  • target_delta (float) – desired fractional change in detector signal between steps
  • backstep (bool) – whether backward steps are allowed – this is concern with some motors
